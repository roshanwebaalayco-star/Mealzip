## PART 1 — BACKEND: One New Route

Add this route to `server/engine/meal-generation-service.ts` alongside the existing routes. Do not create a new router file.

**Route:** `GET /api/meal-plans/:id/day/:date/recipe`

**What it does:** Returns the full enriched recipe data for all three meal slots of a given day, ready for the frontend carousel to render without any further processing.

**Exact implementation steps in order:**

**Step 1 — Auth and validation:**
- Validate `id` (meal plan ID) and `date` (format `YYYY-MM-DD`) from params
- Fetch the meal plan row from Supabase: `SELECT generation_status, days FROM meal_plans WHERE id = :id`
- If `generation_status !== 'completed'`, return HTTP 403 with `{ error: "Recipe viewer is only available after meal generation is complete.", status: <current_status> }`
- This is the gate. Nothing below runs unless status is `completed`.

**Step 2 — Find the day:**
- Parse `days` as `DayPlan[]`
- Find the entry where `day.date === params.date`
- If not found, return 404 with `{ error: "No meal plan found for this date." }`

**Step 3 — Build recipe data for each slot:**

For each slot (`breakfast`, `lunch`, `dinner`):

If the slot is skipped (`meal.skipped === true`): include it in the response with `{ skipped: true, skip_reason: meal.skip_reason, nutritional_bandaid: meal.nutritional_bandaid }` and skip the recipe enrichment below.

If the slot is not skipped:

A. **Recipe lookup:** Check if the meal's `base_recipe` already has `steps[]` and `ingredients[]` populated (non-empty arrays). If yes, use them directly — do not query Gemini.

B. **Database fallback:** If `steps[]` is empty or missing, query the `recipes` table: `SELECT ingredients, steps, prep_time_mins, cook_time_mins FROM recipes WHERE name ILIKE :meal_name LIMIT 1`. If found, use the DB row.

C. **Gemini fallback:** If neither the stored data nor the database has steps, make a Gemini API call. Use the existing `getGeminiClient()` singleton already in `prompt-chain.ts`. The prompt must be:

```
You are a clinical Indian cooking assistant. Generate a standard recipe for "${meal.name}" as served in ${family.stateRegion}.

Return ONLY valid JSON in exactly this shape:
{
  "ingredients": [{ "name": "string", "quantity": "string" }],
  "steps": ["Step 1: ...", "Step 2: ...", "Step 3: ..."],
  "prep_time_mins": number,
  "cook_time_mins": number
}

Rules:
- Do NOT include any health modifications in the steps. Steps are the base recipe only.
- Steps must be numbered: "Step 1:", "Step 2:", etc.
- Use Indian cooking terminology and measurements (katori, tbsp, tsp, grams).
- Maximum 10 steps. Minimum 4 steps.
- Do not add any text outside the JSON object.
```

D. **Modification injection:** This is the critical step. Do NOT skip it. Do NOT delegate it to the frontend.

Take `meal.member_plates[]` — this is the array of `MemberPlate` objects stored in the meal plan. Each has `member_name` and `modifications: string[]`.

Build a `modificationsByStep` map. The algorithm:

```typescript
// Each modification string may contain a step reference like "Step 3" or "step 3"
// or a keyword like "salt", "ghee", "onion", "tempering", "final"
// For each modification string, try to match it to a specific step index.
// If matched: attach it to that step.
// If not matched: attach it to the LAST step (as a general plate instruction).

const STEP_KEYWORDS: Record<string, string[]> = {
  tempering:   ["temper", "tadka", "heat oil", "mustard seed", "jeera", "splutter", "onion", "garlic", "add onion", "fry onion"],
  salt:        ["add salt", "season", "salt to taste", "namak"],
  dairy:       ["add ghee", "add butter", "add paneer", "add curd", "add cream", "add milk", "finish with ghee"],
  sugar:       ["add sugar", "jaggery", "sweetener"],
  serve:       ["serve", "garnish", "plate", "transfer to serving"],
  cook_protein:["add chicken", "add mutton", "add fish", "add egg", "add prawns"],
};

function matchModificationToStep(modification: string, steps: string[]): number {
  // First: explicit step number reference ("Step 3", "step 4", etc.)
  const explicitMatch = modification.match(/step\s+(\d+)/i);
  if (explicitMatch) {
    const stepNum = parseInt(explicitMatch[1], 10);
    if (stepNum >= 1 && stepNum <= steps.length) return stepNum - 1;
  }

  // Second: keyword matching against step text
  const modLower = modification.toLowerCase();
  for (const [, keywords] of Object.entries(STEP_KEYWORDS)) {
    const modHasKeyword = keywords.some((kw) => modLower.includes(kw));
    if (modHasKeyword) {
      const matchedStepIndex = steps.findIndex((step) =>
        keywords.some((kw) => step.toLowerCase().includes(kw))
      );
      if (matchedStepIndex !== -1) return matchedStepIndex;
    }
  }

  // Fallback: attach to last step
  return steps.length - 1;
}
```

Build the final enriched steps array:

```typescript
interface EnrichedStep {
  step_number: number;        // 1-indexed
  instruction: string;        // original step text
  member_modifications: {
    member_name: string;
    modification_text: string;
    urgency: "CRITICAL" | "RECOMMENDED" | "INFO";
    // CRITICAL = contains "ALLERGY CRITICAL", "JAIN RULE", "PULL BEFORE", "DIABETES"
    // RECOMMENDED = contains "HYPERTENSION", "PORTION", "REDUCE", "WITHHOLD"
    // INFO = everything else
  }[];
}
```

Urgency detection: scan the modification string.
- Contains "ALLERGY CRITICAL", "PULL BEFORE", "JAIN RULE", "RELIGIOUS" → `CRITICAL`
- Contains "DIABETES", "KIDNEY", "HYPERTENSION", "SODIUM CAP", "FASTING" → `RECOMMENDED`
- Everything else → `INFO`

**Step 4 — Image URL:**

Check if `meal.base_recipe.image_search_query` exists. Use it as the search term.

Try to find an image using one of these methods in order:

Method A: Check if the meal plan already has a stored image URL for this meal (check `meal.image_url` if it exists in the stored data).

Method B: If `UNSPLASH_ACCESS_KEY` is set in environment, call:
```
https://api.unsplash.com/search/photos?query=${encodeURIComponent(imageQuery + " Indian food")}&per_page=1&orientation=landscape
```
Use the `urls.regular` from the first result.

Method C: If `GOOGLE_CUSTOM_SEARCH_API_KEY` and `GOOGLE_SEARCH_ENGINE_ID` are set, call:
```
https://www.googleapis.com/customsearch/v1?key=${KEY}&cx=${CX}&q=${query}&searchType=image&num=1&imgSize=large
```
Use `items[0].link`.

Method D (fallback — always works): Return `null` for `image_url`. The frontend will show a placeholder. Do not crash if no image API is configured.

**Step 5 — Response shape:**

```typescript
{
  meal_plan_id: string,
  date: string,              // "YYYY-MM-DD"
  day_name: string,          // "Monday"
  meals: {
    breakfast: RecipeSlideData | null,
    lunch: RecipeSlideData | null,
    dinner: RecipeSlideData | null,
  }
}

interface RecipeSlideData {
  slot: "breakfast" | "lunch" | "dinner",
  name: string,              // dish name e.g. "Dal Makhani"
  image_url: string | null,  // from image API or null
  image_query: string,       // what was searched (for frontend fallback)
  estimated_cost_inr: number,
  prep_time_mins: number,
  cook_time_mins: number,
  priority_flags: string[],  // e.g. ["allergy_compliant", "low_sodium"]
  pantry_items_used: string[],
  ingredients: { name: string; quantity: string }[],
  enriched_steps: EnrichedStep[],
  member_plates: {           // pass through for frontend display
    member_name: string,
    modifications: string[],
    fasting_replacement?: string,
    tiffin_instructions?: string,
  }[],
  recipe_source: "stored" | "database" | "gemini_generated",
  skipped: false,
} | {
  slot: "breakfast" | "lunch" | "dinner",
  name: string,
  skipped: true,
  skip_reason: string,
  nutritional_bandaid: string,
}
```

**Error handling for this route:**
- Any error inside the recipe enrichment for one slot must NOT fail the entire response. Wrap each slot in try/catch. If one slot fails, include `{ slot, name, error: "Recipe details unavailable", skipped: false }` for that slot and continue.
- Log the error server-side with `console.error("[RecipeViewer]", error)`.
