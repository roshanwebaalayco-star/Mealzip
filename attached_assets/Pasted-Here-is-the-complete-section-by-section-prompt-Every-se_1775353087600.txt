## SECTION 1 — AUTHENTICATION (Supabase)

1. Signup creates a user in `auth.users` AND creates a corresponding row in the `families` table linked to `auth.users.id`. 5. Logout must call `supabase.auth.signOut()` AND clear all local state (React state, localStorage, sessionStorage). After logout, navigating back must not restore any previous session.

6. Row Level Security on Supabase: Open the Supabase dashboard. Go to every table: `families`, `family_members`, `monthly_budgets`, `weekly_contexts`, `member_weekly_contexts`, `meal_plans`, `grocery_lists`. Confirm RLS is ENABLED on every table. ### 3C — Pre-Generation: The Constraint Packet (CRITICAL — THIS IS WHY SAMOSAS APPEAR)

**This runs entirely in TypeScript before ANY Gemini call. If this is not running, Gemini has no constraints and will suggest whatever it wants.**

Open `conflict-engine.ts`. Verify the function `runConflictEngine()` is being called with the full family data before `runPromptChain()` is called. If `runConflictEngine()` is not called, or is called but its output is not passed to the prompt chain, fix this immediately.

The constraint packet must include:
1. All allergy blocks — every allergen maps to a list of specific Indian ingredient names (peanuts → groundnuts, mungfali, peanut oil, chikki, etc.). These are hard blocks. Gemini must never suggest a meal containing these.
2. All religious/dietary blocks — jain means no onion, garlic, potato, carrot, radish, beetroot, turnip. Sattvic means no onion or garlic. These are hard blocks.
3. All medication interaction constraints — parsed from the free text using regex, not AI.
4. Clinical condition constraints — diabetes means no maida, no refined sugar, no white rice as the primary carb. Hypertension means sodium cap 1500mg/day. Kidney issues means potassium and phosphorus restrictions. These are hard blocks.
5. Budget constraints — the exact daily cost limit per meal slot in INR. Gemini must not suggest a meal that costs more than this.
6. Cooking time constraints — if weekday cooking time is under_20_mins, Gemini cannot suggest a biryani or any dish requiring more than 25 minutes.
7. Regional food constraints — a family in Kerala should get Kerala dishes. A family in Punjab should get Punjabi dishes. The region is passed explicitly. Gemini is not allowed to suggest dishes from a different cuisine unless the family has explicitly set a mixed diet.

**Verify this specific check: open the Gemini prompt being built in `prompt-chain.ts`. Find the section that lists hard constraints. If this section is empty, or if the word "forbidden" or "NEVER" does not appear in the constraints, the constraints are not being enforced. A Gemini prompt that says "prefer healthy options" is not a constraint. A constraint says "NEVER suggest: samosas, pakoras, fried snacks, maida-based foods, refined sugar dishes."**

## SECTION 6 — AI CHAT

The AI chat is not a general chatbot. It has one specific purpose: the user logs a meal they ate that was not in the plan (street food, dining out, a guest's home), and the AI calculates the macro impact and adjusts the week's remaining meals.

**What the AI must do:**
1. User types or speaks what they ate. Example: `"Had 3 samosas and chai at a friend's place for lunch"`.
2. The backend identifies this as an unplanned meal log.
3. The backend estimates macros: samosa (approximately 150 kcal, 8g fat, 18g carbs each × 3 = 450 kcal, 24g fat, 54g carbs). Chai with sugar: 60 kcal. Total unplanned: ~510 kcal.
4. The backend compares this against today's lunch target from the meal plan.
5. The backend adjusts tomorrow's meals to rebalance: if sodium was high (samosas are salty), reduce tomorrow's dinner salt. If fat was high, reduce tomorrow's cooking oil.
6. The AI response must NEVER use these phrases: "I'm sorry", "I apologize", "Great question", "That's okay", "Don't worry". It speaks like a clinical calculator: `"Logged: 510 kcal unplanned at lunch. Tomorrow's dinner adjusted to 420 kcal (down from 520 kcal) to rebalance the week's fat intake. No further change needed to sodium — within weekly limit."`.
7. The adjustment is stored back to `meal_plans.days` in Supabase so the meal plan reflects the change.

**Voice input (Sarvam AI):**
If `SARVAM_API_KEY` is set, enable microphone input. The user speaks in any Indian language. Sarvam transcribes it. The transcription is passed to Gemini for interpretation. If `SARVAM_API_KEY` is not set, show only the text input. Do not crash.
